## Phát Hiện Kỹ Thuật Quan Trọng

### 1. Renderer dùng CPU scanline — bottleneck tiềm ẩn

`DrawFilledCircle()` và `DrawO()` trong [Renderer.cpp](../../CaroGameSDL2/src/sdl/Renderer.cpp) dùng vòng lặp scanline:
```cpp
for (int dy = -radius; dy <= radius; dy++) {
    int dx = sqrt(radius*radius - dy*dy);
    SDL_RenderDrawLine(...);
}
```
**Vấn đề:** Mỗi frame đều tính lại từ đầu dù quân cờ không thay đổi.
**Hướng sửa:** Pre-render lên SDL_Texture một lần khi khởi động → xem DR-001.

### 2. Animation state machine thiết kế tốt

[Animation.cpp](../../CaroGameSDL2/src/sdl/Animation.cpp) dùng `CharState` struct với enum `AnimState`.
Điểm mạnh: mỗi nhân vật độc lập, dễ mở rộng thêm state mới.
Điểm yếu: chưa có squash & stretch, chưa có easing cho movement.

### 3. WinEffect đã có easing functions

`EaseInOutQuad` và `EaseOutBack` được implement tay trong [WinEffect.cpp](../../CaroGameSDL2/src/sdl/WinEffect.cpp).
Đây là kỹ thuật quan trọng — cần hiểu rõ toán học đằng sau:
- `EaseOutBack`: overshoots rồi settle lại → cảm giác "nảy"
- `EaseInOutQuad`: tăng tốc → giảm tốc → tự nhiên
